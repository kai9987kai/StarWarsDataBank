# Code of Conduct

## Our Standards

This project should be a welcoming and practical place to collaborate. Everyone participating in the project is expected to act with respect, patience, and good faith.

Examples of positive behavior include:

- Using clear and respectful language.
- Giving constructive feedback that focuses on the work.
- Accepting reasonable disagreement.
- Helping newcomers understand the project.
- Respecting different experience levels and backgrounds.

Examples of unacceptable behavior include:

- Harassment, threats, insults, or personal attacks.
- Discriminatory language or behavior.
- Sexualized language, imagery, or attention.
- Trolling, repeated disruption, or deliberate intimidation.
- Publishing private information without clear permission.

## Scope

This code of conduct applies to project spaces including issues, pull requests, discussions, documentation, commits, and any other collaboration connected to this repository.

## Reporting

If you see behavior that violates this code of conduct, report it to the project maintainer. Include relevant context such as links, screenshots, dates, and a short description of what happened.

Reports should be handled as privately as practical. Maintainers should avoid sharing identifying information unless it is necessary to review or resolve the issue.

## Enforcement

Maintainers may take action when behavior harms the project or its contributors. Actions may include:

- A private clarification or warning.
- Asking for edits or removal of inappropriate content.
- Temporary limits on participation.
- Permanent removal from project spaces.

Enforcement should be fair, proportionate, and focused on keeping the project productive and respectful.

## Maintainer Responsibilities

Maintainers are responsible for clarifying standards, responding to reports, and applying this code of conduct consistently. Maintainers should not ignore harmful behavior because it comes from an active contributor or long-time participant.

## Attribution

This code of conduct is a plain-language project policy inspired by common open source community standards.
